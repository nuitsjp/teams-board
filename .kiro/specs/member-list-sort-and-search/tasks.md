# Implementation Plan

- [x] 1. メンバー一覧のソートロジックを名称昇順に変更する
- [x] 1.1 ソート基準を勉強時間降順から名称昇順に切り替える
  - 既存のソート処理をロケール対応の文字列比較に置き換える
  - 日本語名・英語名が混在してもあいうえお順・アルファベット順で正しく並ぶようにする
  - _Requirements: 1.1, 1.2, 1.3_

- [x] 2. 検索入力フィールドをヘッダーに追加する
- [x] 2.1 検索用のローカル状態を導入し、ヘッダー内に検索ボックスを配置する
  - コンポーネントに検索クエリのローカル状態を追加する
  - 「メンバー」見出しの右側、メンバー数バッジの左側に検索入力フィールドを表示する
  - 検索アイコン付きの入力フィールドとし、プレースホルダーに「名前で検索...」を設定する
  - 既存のヘッダーデザインと一貫したスタイルを適用する
  - _Requirements: 2.1, 2.2, 2.3_

- [x] 3. インクリメンタルサーチによる絞り込み機能を実装する
- [x] 3.1 検索クエリによるフィルタリングと動的件数表示を実装する
  - 検索フィールドへの入力に応じて、名前の部分一致でメンバーを絞り込む
  - 大文字・小文字を区別せずに検索する
  - 検索フィールドが空の場合は全メンバーを表示する
  - 絞り込み後もソート順（名称昇順）を維持する
  - メンバー数バッジに絞り込み後の件数を反映する
  - _Requirements: 3.1, 3.2, 3.3, 3.5, 3.6_

- [x] 3.2 検索結果が0件の場合のメッセージを表示する
  - フィルタ結果が空の場合に「該当するメンバーが見つかりません」メッセージを表示する
  - メッセージはリスト領域内に適切なスタイルで表示する
  - _Requirements: 3.4_

- [x] 4. コンポーネントテストを作成する
- [x] 4.1 ソートと検索フィルタリングの動作を検証するテストを追加する
  - メンバーが名称昇順で表示されることを検証する
  - 検索フィールドに文字を入力するとメンバーが絞り込まれることを検証する
  - 検索をクリアすると全メンバーが表示に復帰することを検証する
  - 検索結果0件時にメッセージが表示されることを検証する
  - バッジ件数が絞り込み結果に連動することを検証する
  - _Requirements: 1.1, 1.3, 3.1, 3.2, 3.3, 3.4, 3.5, 3.6_
