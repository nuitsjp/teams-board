# 参加者レポートの取込とグループ開催実績の確認

管理者が Microsoft Teams の参加者レポートをシステムに取り込み、閲覧者が特定グループの開催実績（開催回数・総参加時間・セッション別参加者）を確認するシナリオ。

!!! info
    本シナリオは管理者による取り込みと閲覧者による確認のコラボレーションを記述している。グループの活動頻度・参加者推移・継続性を把握することで、活動の評価や改善に活用できる。

## 前提条件

- 管理者が Microsoft Teams 管理センターにアクセスできる
- 管理者が SAS トークン付き URL でシステムにアクセスできる
- 確認対象のグループが過去に1回以上のセッションを開催している

## シナリオ

| No. | アクター | アクティビティ | 説明 |
|--|--|--|--|
| 1 | 管理者 | 参加者レポート取得 | Microsoft Teams 管理センターから対象会議の参加者レポートをダウンロードする |
| 2 | 管理者 | 参加者レポート登録 | 管理者パネルを開き、参加者レポートをドロップゾーンに投入する。システムがレポートを解析し、解析結果を確認して一括保存する |
| 3 | 閲覧者 | グループ開催実績閲覧 | グループ一覧から確認対象のグループを選択し、セッション別の参加者と開催実績を確認する |

## 事後条件

- 新規セッション情報が保存されている
- インデックスにグループの開催履歴が反映されている
- ダッシュボードに最新の統計情報が表示されている
- グループ詳細画面でセッション別の参加者情報が確認できる

## 制約

なし
